# Protocolos Globais de Agente (MCP Toolkit)

> Você possui ferramentas MCP personalizadas para automação de fluxo de trabalho. Siga estes protocolos rigorosamente em qualquer workspace:

## 1. Persistência Contínua (Silent Logger)
- **Ação**: A CADA resposta finalizada ou passo significativo, invoque a tool `silent_logger`.
- **Objetivo**: Garantir que o contexto seja salvo no Project Journal local em `./docs/journal.md`. Se a pasta `docs` não existir, crie-a.

## 2. Fluxo de Commit "Smart" (Strict Git Commit Agent)
- **Gatilho**: Quando houver mudanças prontas para commit ou a meu pedido.
- **Processo**:
  1. Identifique o **Project ID** no arquivo `GEMINI.md` (ou contexto atual). Se não houver, PERGUNTE AO USUÁRIO.
  2. Invoque a tool `git_commit_agent` preenchendo os campos obrigatórios:
     - `project_id`: (ex: GEMINI-TOOLKIT)
     - `commit_type`: (func, fix, refactor, test, docs, style, build)
     - `commit_title`: Resumo curto.
     - `commit_description`: Descrição detalhada.
  3. A tool cuidará da formatação final da mensagem. Mensagens de texto livre não são mais aceitas.

## 3. Recuperação de Sessão (Reboot)
- **Gatilho**: Se eu digitar "reboot" ou iniciar um chat mencionando que a sessão caiu.
- **Ação**: Leia imediatamente o conteúdo de `./docs/journal.md`.
- **Comportamento**: Restaure sua consciência situacional com base no "Consolidated State" e no histórico de decisões registrados no arquivo antes de prosseguir.
